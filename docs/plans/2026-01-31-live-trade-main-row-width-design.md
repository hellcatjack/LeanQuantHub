# 实盘交易主行卡片宽度调整设计

## 背景与问题
当前“交易状态 / 账户概览 / 当前持仓”三卡同排展示时，交易状态与账户概览在部分分辨率下出现文案超出边框的问题。用户明确要求将交易状态与账户概览宽度调整为 280px，以消除溢出并保持布局稳定。

## 目标与范围
- 目标：将“交易状态 / 账户概览”两张卡在桌面布局中固定为 280px，避免文字溢出。
- 范围：仅调整 `live-trade` 页面主行网格布局，不改变其他卡片的内容、样式或刷新逻辑。
- 非目标：不改动移动端单列布局，不引入新的 UI 交互或信息结构变更。

## 方案对比
1) **固定宽度 280px（推荐）**
   - 优点：满足用户明确要求，布局稳定，文案溢出风险最小。
   - 缺点：在超宽屏上可用空间更多时，两张卡不会再扩大。

2) **区间宽度 minmax(280px, 320px)**
   - 优点：在宽屏上更舒展；仍有最小宽度保障。
   - 缺点：与“固定 280px”要求不完全一致，可能引起验收争议。

3) **使用 CSS 变量驱动可配置宽度**
   - 优点：未来可配置；便于多端适配。
   - 缺点：超出本次调整范围，增加复杂度。

## 决策
采用方案 1：在桌面布局（`min-width: 1200px`）下将前两列固定为 280px，第三列保留弹性 `minmax(360px, 1fr)`，确保“当前持仓”占用剩余空间。

## 实现要点
- 修改 `frontend/src/styles.css` 中 `.live-trade-main-row` 的桌面断点布局：
  - `grid-template-columns: 280px 280px minmax(360px, 1fr);`
- 保留小屏布局为自动折行（`repeat(auto-fit, minmax(280px, 1fr))`）。

## 测试策略
- Playwright：新增/调整断言，验证桌面布局下两张卡宽度 >= 280px，且“当前持仓”宽度最大。
- 手动：在 1200px 以上桌面窗口查看文案是否溢出。
