# ID 展示规范化（ID Chip）设计

## 背景
实盘交易与数据页存在需要手工填写的 ID（项目/快照/批次/订单/任务等），但 UI 分散、难以获取。需要统一展示规范，降低操作成本与误填风险。

## 目标
- 统一 ID 展示样式（ID Chip + 复制按钮）。
- 覆盖实盘交易、数据页、项目页、回测/报告页等关键位置。
- 提供可复制的纯数字 ID。

## 非目标
- 不改动 ID 的业务含义或生成逻辑。
- 不引入新的后端接口（优先利用现有字段）。

## 统一样式
- 格式：`类型#ID`（例：`Project#16`、`Snapshot#91`、`Run#123`）。
- 组件：`IdChip(label, value, copy)`，右侧复制按钮（tooltip：复制 ID）。
- 展示位置：表格 ID 列/名称旁/卡片 meta 区。
- 拥挤时：仅显示 1–2 个核心 ID，其余折叠到“更多”。

## 范围与优先级
1) **实盘交易页**：项目绑定卡片、执行上下文、批次列表、订单/成交明细。
2) **数据页**：Pretrade 运行区、历史任务、步骤列表、PIT/基础面任务。
3) **项目页**：决策快照与训练/回测关联 ID。
4) **回测/报告页**：回测 Run ID / 报告 Run ID。

## 数据映射原则
- 有 ID 列则用 `IdChip` 替换纯文本。
- 无 ID 列则在标题/名称旁追加 `IdChip`。
- 复制内容为纯数字 ID，避免混淆。

## 测试策略（TDD）
- 单元：`IdChip` 组件渲染与复制行为。
- Playwright：LiveTrade 与 DataPage 各添加 1 个 ID 可见性测试。

## 风险与权衡
- 表格密度提升：通过折叠与限制展示数量缓解。
- ID 类型多样：统一命名与顺序，避免信息噪声。
