# 实盘交易-当前持仓刷新摘要与紧凑行高设计

## 背景
当前持仓卡片展示“刷新间隔/下次/上次”三行信息，用户反馈数字逻辑不清晰且与 1 分钟间隔不一致；同时持仓表格行高过大，操作按钮偏高，影响信息密度。

## 目标
- 将当前持仓的三行刷新信息压缩为一行摘要，减少纵向占位并提升可读性。
- 明确区分“刷新调度时间”和“数据更新时间”。
- 缩小持仓表格行高与行内操作按钮高度，提升密度。

## 方案
1) **刷新摘要一行化（A）**
   - 展示格式：`刷新 · 刷新间隔 · 上次刷新 · 下次刷新`，统一在一行显示。
   - 数据更新时间保留为独立一行（避免与调度刷新混淆）。
   - 使用统一的 `renderRefreshSchedule` 增加 `inline` 变体，保证仅当前持仓使用一行布局。

2) **刷新逻辑修正**
   - 对“下次刷新”显示逻辑：优先使用 `lastAt + intervalMs` 计算，保证展示的间隔与配置一致，避免出现非 1 分钟节奏的误解。

3) **表格与操作控件紧凑化**
   - 当前持仓表格增加 `positions-table` 样式，缩小 `th/td` 的 padding 与字号。
   - 行内操作容器使用 `positions-action-group`，缩小间距。
   - 行内数量输入与按钮使用 `positions-action-input` / `positions-action-button`，将高度降至 22px（必要时覆盖默认控件高度）。

## 影响范围
- 仅影响 Live Trade 的当前持仓卡片与其表格样式。
- 其他表格与按钮不受影响（通过专用 class 限定）。

## 测试策略
- Playwright：新增断言
  - `card-refresh-inline-positions` 可见（刷新摘要一行化）
  - `positions-table` / `positions-action-input` / `positions-action-button` class 存在
  - 输入框与按钮高度 <= 22px（不降低字体大小）
- 手动：检查当前持仓卡片
  - 刷新摘要为单行
  - 数据时间仍单独显示
  - 行高与操作按钮明显变紧凑
