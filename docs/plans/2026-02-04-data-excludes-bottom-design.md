# 数据页全局排除列表下移设计

## 背景
数据页当前将“全局排除列表”置于页面中部，用户希望其出现在整页最下方，以便与其他数据同步/质量/PIT 相关区块形成清晰层级。

## 目标
- 将“全局排除列表”移动到 DataPage 页面最底部。
- 不改变现有功能、接口与交互行为。

## 设计方案
- 仅调整 `frontend/src/pages/DataPage.tsx` 渲染顺序：将全局排除列表 `<div className="card">` 区块移出当前 `data-columns` 中部位置，放到 DataPage 的最后一个卡片之后。
- 逻辑与状态不变（加载、编辑、启停逻辑、提示文案保持原样）。
- 增加 Playwright 用例检查“全局排除列表”处于页面卡片标题列表的最后一项，避免回归。

## 风险与影响
- 仅布局顺序变更，API 与状态不变，风险低。
- 若页面后续新增底部区块，需要同步调整该“最后位置”的断言。

## 验收标准
- 数据页滚动到底部可见“全局排除列表”。
- 功能完整（新增/编辑/禁用/启用仍可操作）。
- Playwright 测试通过。
