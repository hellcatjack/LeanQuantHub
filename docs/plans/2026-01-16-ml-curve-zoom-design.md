# ML 训练曲线可读性优化设计

## 目标
- 修正训练曲线图表字体偏大问题。
- 提升验证曲线在“平台化/变化很小”时的可读性。
- 不引入新图表库，沿用现有 SVG 渲染。

## 设计要点
### 1) 双坐标轴 + 验证曲线放大
- 左轴：继续使用训练+验证的全局 min/max（保持绝对尺度）。
- 右轴：仅针对验证曲线（valid）的 min/max 自动放大。
- 右轴刻度数量 3–4 个，并使用更小字号标注。
- 验证曲线用右轴尺度绘制，避免“贴平”。

### 2) 视觉与标注
- 坐标轴标签字体缩小至 10px。
- 验证线条改为更细线 + 虚线（保持辨识度）。
- 最优点标签字体缩小并上移，避免遮挡。
- 图例新增“Valid（放大）”标注，明确右轴含义。

## 缩放策略
当 valid 曲线跨度过小时：
- 以 `max(|min|, |max|, 1)` 作为基准值，设定最小跨度（例如 2%）。
- 额外添加 10% padding，避免贴边。

## 影响范围
- `frontend/src/pages/ProjectsPage.tsx`（renderMlCurve）
- `frontend/src/styles.css`（.ml-curve-*）
- `frontend/src/i18n.tsx`（新增/调整文案）

## 验收标准
- 字体明显缩小，轴标签不再显得“抢眼”。
- Valid 曲线可见细微起伏，并在右轴有清晰刻度。
- 图例明确标注“Valid（放大）”，不产生误读。
