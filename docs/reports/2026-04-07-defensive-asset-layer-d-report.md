## 观察

### 1. `QQQ/SOXX` benchmark 会让系统更少进入 risk-off
- D0 的 `RiskOff Count=2`，而 D1/D2 都只有 `1`。
- 手续费和换手率也随之下降。
- 这说明科技和半导体 benchmark 更容易让系统保持 risk-on，而不是更频繁地收缩风险。

### 2. 全样本效率略有改善，但不是无条件更稳
- D1/D2 的 CAGR、Sharpe、全样本 MaxDD 都略优于 D0。
- 但这并不意味着“QQQ/SOXX 更适合默认 benchmark”，因为分阶段稳定性并没有一致改善。

### 3. 2022 和 2023 的阶段回撤变差
- 2022：
  - D0：`-2.62% / -7.55%`
  - D1：`-0.42% / -9.80%`
  - D2：`-0.41% / -9.80%`
- 2023-2024：
  - D0：`20.49% / -3.47%`
  - D1：`18.92% / -5.49%`
  - D2：`18.87% / -5.39%`
- 这表明 `QQQ/SOXX` benchmark 虽然让过滤器更少触发，但也削弱了部分阶段的保护质量。

### 4. `QQQ` 与 `SOXX` 的行为几乎重合
- 两者的阶段结果、滚动风险和风险关闭次数几乎一致。
- 这说明它们在这里主要扮演的是“更高 beta 的市场温度计”，而不是提供了本质不同的过滤逻辑。

### 5. 这些 benchmark 更适合作为敏感度测试，而不是默认值
- 它们的收益效率略好，但代价是过滤器更偏向继续暴露风险。
- 在“稳健优先”的长期目标下，这种 trade-off 还不足以替代 `SPY` 作为默认基准。

## 按稳健门槛判断
按实验计划里的“稳健优先”门槛：
- D1/D2 没有形成足够强的跨阶段稳健优势，不应替代 `SPY` 成为默认 benchmark。
- `QQQ/SOXX` 更适合作为风险对照和敏感度实验，用来检测过滤器是否过度依赖高 beta 资产。

## Layer D 结论
1. **默认 benchmark 继续保持 `SPY`。**
2. **`QQQ` 和 `SOXX` 只保留为风险对照与敏感度测试，不作为默认 benchmark。**
3. **若未来要考虑切换默认 benchmark，必须证明它在多个压力阶段的回撤控制优于 `SPY`，而不是仅仅提升全样本收益效率。**
