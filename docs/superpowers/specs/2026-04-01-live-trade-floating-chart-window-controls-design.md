# 实盘交易浮窗图表拖动吸附与最小化设计

## 目标
在现有单行持仓表基础上，将专业图表浮窗升级为可拖动、可吸附、可记忆位置与尺寸、可最小化的轻量终端面板，同时保持选中标的避让与窄屏回退行为。

## 问题
1. 当前浮窗位置完全由选中行自动计算，用户无法主动调整。
2. 浮窗大小固定，无法适配不同监控习惯。
3. 没有最小化，图表即使暂时不用也持续占用表格右侧空间。
4. 重新刷新页面后，用户之前调整过的位置和大小会丢失。

## 方案
### 桌面端浮窗状态模型
浮窗维护四类状态：
- `floating`: 当前是否处于桌面浮窗模式
- `minimized`: 是否折叠为标题条
- `anchor`: 吸附边缘，取值 `top-right`、`bottom-right`、`top-left`、`bottom-left`
- `size`: 浮窗宽高，允许在设定范围内调整

### 拖动与吸附
- 拖动手柄放在图表头部，仅桌面端启用。
- 拖动结束时根据浮窗中心点距离最近的卡片边缘决定吸附象限。
- 吸附后保留与边缘的偏移量，防止每次都硬贴边。
- 当没有用户手动拖动记录时，仍沿用当前“避开选中行”的自动布局。

### 尺寸调整
- 桌面端右下角提供 resize 手柄。
- 宽高均限制在安全区间内，防止遮挡整个表格。
- 调整尺寸后重新约束到卡片边界内，并保留当前吸附边缘。

### 最小化
- 最小化后浮窗收缩为一条标题条，仅保留：symbol、周期、恢复按钮。
- 最小化状态仍可拖动和吸附。
- 还原后恢复上一次记忆的尺寸和吸附位置。

### 状态持久化
- 使用 `localStorage` 按页面维度保存：最小化状态、吸附边缘、偏移量、宽高。
- 仅桌面端读取和写入；窄屏回退时不覆盖桌面状态。
- 若保存值越界或损坏，自动回退到系统布局计算值。

### 窄屏策略
- 视口小于桌面阈值时继续使用表格下方堆叠图表。
- 拖动、缩放、吸附、最小化控件全部隐藏。

## 关键实现
1. `LiveTradePage.tsx`
   - 新增浮窗持久化状态、拖动/缩放计算、边界约束与吸附解析纯函数。
   - 浮窗布局优先使用用户手动状态；无手动状态时回到自动避让布局。
2. `PositionChartWorkspace.tsx`
   - 增加浮窗头部控制区、拖动手柄、最小化按钮、尺寸手柄。
   - 最小化时仅渲染精简头部。
3. `styles.css`
   - 增加浮窗拖动态、最小化态、尺寸手柄、吸附态样式。
4. 测试
   - 单测覆盖：吸附象限解析、持久化值修复、最小化布局解析。
   - Playwright 覆盖：拖动后位置变化、刷新后位置记忆、最小化/还原、尺寸调整。
