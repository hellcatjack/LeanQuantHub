# 实盘交易当前持仓单行压缩与浮窗图表设计

## 目标
将实盘交易页面的当前持仓压缩为高密度单行表格，避免每个标的占 3 行；专业图表改为桌面端浮窗，不再挤压表格横向空间，并自动避开当前选中标的所在行与左侧关键列。

## 现状问题
1. 当前持仓使用并排双栏布局，表格与图表同时占据横向空间。
2. 每行操作区为两层面板，直接把表格行高拉大。
3. 图表作为并排区域时，桌面端可读性下降；在中宽屏会过早切成上下布局。

## 方案
### 桌面端
- 当前持仓卡片继续占满整行。
- 表格区域全宽展示，操作列改为单行工具条：数量、时段、类型、限价、买入、卖出、平仓。
- 专业图表改为浮窗卡片，绝对定位在当前持仓卡片内部右侧。
- 浮窗根据选中标的所在行自动选择放在该行上方或下方，保证不遮挡该行。
- 浮窗左边界保持在右侧区域，避免遮挡 Symbol 等关键列。

### 中窄屏
- 视口不足时自动退回静态块状图表，放在表格下方，不使用浮窗。

## 关键实现
1. 在 `LiveTradePage.tsx` 中为持仓卡片、表格区域和每个持仓行建立 ref。
2. 增加纯函数 `resolvePositionChartFloatingLayout()`，负责根据卡片尺寸、视口尺寸和选中行位置计算浮窗布局。
3. `PositionChartWorkspace.tsx` 支持 `floating` 样式和布局 class。
4. `styles.css` 调整表格、操作列和浮窗样式；操作列单行化，图表浮窗桌面端绝对定位，窄屏端回退为静态布局。

## 验证
- Vitest：覆盖浮窗布局计算逻辑。
- Playwright：覆盖桌面端单行行高、浮窗避让选中行、窄屏堆叠回退。
- `npm run build` 与 `systemctl --user restart stocklean-frontend`。
