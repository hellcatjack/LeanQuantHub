# 稳健期权收益增强研究设计

## 1. 背景

当前 StockLean 的默认长期目标已经明确为“稳健优先于收益最大化”。在这个前提下，用户希望继续调研如何利用期权更加稳健地增加收益。

本设计不直接引入真实期权交易功能，而是先定义一套严谨、可复现、可量化的研究路径，用于回答两个问题：

1. 期权收益增强是否真的能在本项目中提升长期稳健性，而不是单纯抬高短期收益。
2. 如果答案是肯定的，后续应该优先建设哪一类实现路径。

## 2. 当前系统现实边界

### 2.1 已有能力

- Lean + IB 底层具备期权相关基础能力。
- 系统已有稳定的日线股票/ETF 数据、回测框架、项目配置、风险开关和实盘/虚拟盘执行体系。
- 本地已具备若干 option-income / buy-write ETF 的复权日线数据，可用于代理研究：
  - `XYLD`
  - `QYLD`
  - `JEPI`
  - `JEPQ`
  - `DIVO`

### 2.2 关键缺口

- 应用层交易链路目前仍是股票路径，未暴露期权链、期权合约选择、Greeks、IV、行权/指派处理。
- 数据层没有现成可用的期权链历史、Greeks 历史、IV 历史、bid/ask、open interest 数据集。
- 当前账户与策略结构偏向“小账户 + 多标的 + 单票低仓位”，天然不适合直接做标准 covered call。

### 2.3 直接结论

当前阶段不应把“真实期权卖方交易”直接接入主交易链路。更稳健的顺序是：

1. 先用 option-income / buy-write ETF 做代理研究。
2. 只有代理研究证明对稳健性有净提升，才进入真实期权交易链路设计。

## 3. 研究目标

本研究的唯一目标不是“提高收益”，而是评估下列更严格的目标是否成立：

- 在最大回撤恶化不超过 `1.5` 个百分点的前提下，提高组合长期收益效率。
- 在压力阶段的恢复时间恶化不超过基线 `20%` 的前提下，保持净值曲线稳定性。
- 在默认稳健基线 `SGOV / SGOV,VGSH / SPY` 的框架下，找到可接受的“收益增强 sleeve”。

## 4. 研究对象分层

### 4.1 Core Proxy Layer

作为期权收益增强的代理资产，先研究：

- `JEPI`
- `JEPQ`
- `XYLD`
- `QYLD`
- `DIVO`

这些资产不是“真实期权策略复现”，但它们能较低成本地回答：

- 期权收益增强风格是否适合本项目的稳健目标
- 哪类收益增强更偏稳健，哪类只是牺牲上行换分红

### 4.2 Defensive Integration Layer

研究方式不是把上述代理资产当成主策略替代品，而是作为“收益增强 sleeve”插入现有稳健框架中。优先研究两种集成方式：

1. 替代部分防御仓
2. 替代部分闲置现金 / defensive idle

禁止的首轮做法：

- 直接替代全部防御仓
- 直接替代主 alpha 风险仓

### 4.3 Future Real Options Layer

只有当代理研究通过门槛后，才进入真实期权策略研究。首选候选顺序固定为：

1. 受限风险、单腿、规则简单的收益增强策略
2. 真实 covered call / cash-secured put 的数据与执行可行性评估
3. collar 或更复杂策略

当前阶段不设计多腿组合自动交易。

## 5. 方案选择

### 方案 A：直接建设真实期权交易

优点：
- 理论上最接近期权收益增强目标

缺点：
- 当前系统和数据都不具备生产级稳健条件
- 会同时引入数据、执行、风控、指派、UI 五类复杂度
- 风险最高

结论：不推荐作为第一阶段。

### 方案 B：先做代理 ETF 研究，再决定是否建设真实期权

优点：
- 可以复用现有数据和回测框架
- 研究成本低，反馈快
- 更符合“稳健优先”

缺点：
- 代理结果不能完全替代真实期权执行结论

结论：推荐。

### 方案 C：只做理论分析，不做系统化实验

优点：
- 成本最低

缺点：
- 不能形成工程可执行结论
- 容易再次陷入主观判断

结论：不推荐。

## 6. 推荐方案

采用方案 B，分三阶段推进：

### 阶段 1：代理研究

围绕 `JEPI/JEPQ/XYLD/QYLD/DIVO` 建立期权收益增强代理矩阵，评估其对默认稳健框架的影响。

### 阶段 2：稳健性门槛判定

若某个代理组合满足稳健门槛，再进入下一阶段；若不满足，则明确终止真实期权建设。

### 阶段 3：真实期权可行性设计

只对通过阶段 2 的路径，继续设计真实期权数据、筛选、执行和风控链路。

## 7. 实验矩阵

### 7.1 基线

- Default Baseline:
  - `risk_off_symbol = SGOV`
  - `risk_off_symbols = SGOV,VGSH`
  - `benchmark = SPY`

### 7.2 Sleeve 配置

首轮研究以下代理 sleeve：

- `JEPI`
- `JEPQ`
- `XYLD`
- `QYLD`
- `DIVO`

### 7.3 配置方式

每个代理资产都做两类实验：

1. 替代部分防御仓
   - 例如 `20%`、`30%` 的 defensive sleeve
2. 替代部分 idle 配置
   - 保留风险仓不变，仅替换 idle 停放路径

### 7.4 明确不做

- 不把代理资产当成新的 benchmark
- 不直接替代主风险仓
- 不把高收益代理直接纳入默认值

## 8. 评估指标

必须同时评估收益与稳健性，且稳健性优先级更高。

### 8.1 收益效率

- Total Return
- CAGR
- Sharpe
- Sortino

### 8.2 稳健性

- Max Drawdown
- Ulcer Index
- 恢复时间
- 压力阶段净值跌幅
- 年度收益稳定性

### 8.3 行为一致性

- `risk_off` 触发频率是否被破坏
- 防御资产切换逻辑是否仍可解释
- 是否显著放大风格漂移

## 9. 决策门槛

默认要求如下：

1. 相对默认基线，`Max Drawdown` 恶化不得超过 `1.5` 个百分点。
2. 相对默认基线，`Recovery Time` 恶化不得超过 `20%`。
3. 相对默认基线，`Ulcer Index` 恶化不得超过 `10%`。
4. 收益改善至少满足以下之一：
   - `CAGR` 提升不少于 `0.5` 个百分点
   - `Sharpe` 提升不少于 `0.05`
5. 若收益改善未达门槛，但稳健性恶化，直接淘汰。
6. 只有当“收益改善达标 + 稳健性门槛全部通过”同时成立，才允许进入真实期权阶段。

上述阈值是本轮研究的硬门槛，不允许在报告阶段做主观豁免。

## 10. 输出物

阶段 1 研究完成后，必须产出：

- 分组回测结果表
- 压力阶段对比
- 稳健性排序
- 是否进入真实期权设计的明确结论

## 11. 后续实施边界

若未来进入真实期权阶段，必须新增并补齐：

- 期权链获取接口
- 合约筛选规则
- IV / Greeks / 流动性数据
- 行权/指派处理
- 除息与事件风险控制
- UI 期权合约选择与审计显示
- paper-only 验证阶段

在这些能力补齐前，不允许把真实期权执行接入默认主路径。

## 12. 验收标准

本设计完成的验收标准是：

1. 系统内有一份固定的、可复用的期权收益增强研究规范。
2. 后续实验默认以“稳健优先”解释结果。
3. 真实期权建设不会跳过代理研究阶段。
